# Реалізація алгоритму Rijndael (AES-128) у режимі OFB

Даний проект є лабораторною роботою з дисципліни «Криптологія», що присвячена реалізації симетричного блочного шифру Rijndael (Advanced Encryption Standard) у режимі зворотного зв'язку по виходу (Output FeedBack) мовою Java.



## 1. Теоретичні відомості

### Алгоритм Rijndael (AES-128)
Rijndael — це ітеративний блочний шифр, який оперує матрицею байтів розміром 4x4, що називається станом (State). Для версії з довжиною ключа 128 біт алгоритм виконує 10 раундів перетворень.

Кожен раунд (крім останнього) складається з чотирьох основних трансформацій:
1. **SubBytes**: Нелінійна заміна кожного байта за допомогою таблиці замін (S-box).
2. **ShiftRows**: Циклічний зсув рядків матриці стану на різну кількість байтів (0, 1, 2 та 3 відповідно).
3. **MixColumns**: Перемішування даних у стовпцях шляхом множення матриці стану на фіксований многочлен у полі Галуа $GF(2^8)$.
4. **AddRoundKey**: Накладання раундового ключа на поточний стан за допомогою операції XOR.

У фінальному раунді етап **MixColumns** пропускається.

### Режим OFB (Output FeedBack)
Режим зворотного зв'язку по виходу перетворює блочний шифр на потоковий. У цьому режимі алгоритм Rijndael шифрує не сам текст, а вектор ініціалізації (IV). Результат шифрування стає «гаммою» (keystream), яка накладається на відкритий текст через XOR, і одночасно стає входом для шифрування наступного блоку.

Перевагою OFB є те, що помилка в одному біті шифротексту при передачі не поширюється на наступні блоки при розшифруванні.

## 2. Опис реалізації

Програмний комплекс складається з трьох основних модулів:

### Клас `Rijndael.java`
Це ядро системи, де реалізовано:
- **SBOX та RCON**: Статичні масиви для заміни байтів та раундових констант.
- **keyExpansion**: Метод, що розгортає початковий 128-бітний ключ у 11 раундових ключів (всього 44 слова).
- **encryptBlock**: Реалізація повної послідовності раундів (SubBytes, ShiftRows, MixColumns, AddRoundKey) для обробки одного 16-байтового блоку.
- **mul**: Допоміжний метод для множення чисел у полі Галуа, необхідний для етапу MixColumns.

### Клас `Rijndael_OFB.java`
Цей клас реалізує логіку потокового шифрування:
- Метод **process** виконує ітеративне шифрування блоків IV та операцію XOR з вхідними даними. Завдяки властивостям режиму OFB, цей самий метод використовується і для зашифрування, і для розшифрування.
- Метод підтримує роботу з даними довільної довжини, обробляючи залишки тексту, що менші за розмір блоку (16 байт).

### Клас `Main.java`
Виконує роль інтерфейсу користувача та координатора тестування. Містить демонстраційні вектори (ключ та IV) та логіку взаємодії з консоллю.

## 3. Тестування та перевірка роботи

Для перевірки коректності реалізації у програмі передбачено кілька сценаріїв:

1. **Тест короткого повідомлення**: Перевірка роботи алгоритму на тексті, довжина якого менша за один блок (наприклад, "Hello!"). Це підтверджує правильність обробки неповних блоків у режимі OFB.
2. **Тест довгого повідомлення**: Шифрування речення, що займає кілька блоків. Це демонструє коректність ітерацій та оновлення вектора ініціалізації між блоками.
3. **Аналіз впливу IV (Test 3)**: Шифрування однакового тексту з різними векторами ініціалізації. Результат показує, що зміна IV призводить до повної зміни шифротексту, що є критично важливим для безпеки.
4. **Інтерактивна перевірка**: Користувач може ввести будь-який рядок, який програма зашифрує в Hex-формат та одразу розшифрує назад, демонструючи цілісність даних.

Всі виводи в консоль здійснюються англійською мовою для відповідності міжнародним стандартам розробки, тоді як внутрішня документація та коментарі виконані українською мовою.
