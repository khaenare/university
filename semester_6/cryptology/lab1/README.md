# Реалізація підпису Шнорра

Реалізація криптографічного алгоритму цифрового підпису Шнорра мовою Java.

**Проєкт виконано в навчальних цілях для лабораторної роботи з дисципліни Криптологія студентом групи ПІ-31 Ляшенко Матвієм.**

## Про алгоритм

Схема Шнорра — алгоритм електронного цифрового підпису, безпека якого базується на складності обчислення дискретного логарифма. Він відзначається високою ефективністю, швидкістю обчислень та відносно короткою довжиною підпису порівняно з іншими стандартами.

## Принцип роботи

Алгоритм складається з трьох основних етапів:

1. **Генерація параметрів і ключів**
   Обираються загальнодоступні параметри:
   - Велике просте число p.
   - Просте число q, яке є дільником p - 1.
   - Елемент g порядку q за модулем p.
   
   Генеруються ключі:
   - Приватний ключ x: випадкове число в діапазоні (0 < x < q).
   - Публічний ключ y: обчислюється як y = g^x mod p.

2. **Підписання повідомлення**
   Для заданого повідомлення M відправник:
   - Обирає випадкове число k в діапазоні (0 < k < q).
   - Обчислює параметр r = g^k mod p.
   - Обчислює першу частину підпису e = H(M || r), де H — криптографічна хеш-функція (у даній реалізації SHA-256).
   - Обчислює другу частину підпису s = (k + x * e) mod q.
   
   Електронним підписом повідомлення M є пара значень (e, s).

3. **Перевірка підпису (Верифікація)**
   Одержувач для перевірки достовірності підпису (e, s) виконує наступне:
   - За допомогою публічного ключа y обчислює перевірочне значення r' = (g^s * y^(-e)) mod p.
   - Перевіряє рівність e == H(M || r').
   
   Якщо значення ідентичні, підпис вважається дійсним. Це гарантує цілісність даних та автентичність відправника.

## Запуск та тестування

Для перевірки роботи алгоритму виконайте наступні команди у терміналі:

1. Компіляція вихідного коду:
   `javac Schnorr.java Main.java`

2. Запуск програми:
   `java Main`

Програма згенерує ключі, запропонує ввести текст для підпису, виведе значення підпису (e, s) та автоматично проведе верифікацію для оригінального та зміненого тексту, щоб продемонструвати захист від підміни даних.
