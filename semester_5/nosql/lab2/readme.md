# Лабораторна робота 2: Використання NoSQL та SQL баз даних

## Опис роботи

Ця лабораторна робота була спрямована на порівняння та інтеграцію двох типів баз даних — **NoSQL (MongoDB)** та **SQL (PostgreSQL)**, зокрема для роботи з платформою для фріланс-проєктів. Ми реалізували зберігання даних у обох типах баз, порівняли їх швидкодію та розглянули, як кожен тип баз даних може бути використаний для різних частин системи.

## Кроки виконання

### 1. Ознайомлення з видами NoSQL баз
Ми використали **MongoDB** як NoSQL базу даних. **MongoDB** є документною базою, яка зберігає дані у вигляді BSON-документів. Це дозволяє працювати з неструктурованими та гнучкими даними, що є корисним у нашій предметній області.

### 2. Визначення, чи буде ефективніше винести частину даних в NoSQL
У нашій предметній області, де йдеться про фріланс-платформу, **MongoDB** підходить для зберігання **неструктурованих** даних, таких як повідомлення, пропозиції фрілансерів і проекти, де структура даних може змінюватися. Водночас для структурованих даних (наприклад, платіжні транзакції) **PostgreSQL** виглядає більш доцільним через свою сильну підтримку транзакцій.

### 3. Перенос частини даних у NoSQL базу
Ми перенесли такі дані в **MongoDB**:
- **Проекти**
- **Пропозиції фрілансерів**
- **Повідомлення між користувачами**

Ці дані добре підходять для **NoSQL** баз, оскільки їх структура може змінюватися і немає потреби в суворих відносинах між ними.

#### Файл: `filling_nsql.py`
У цьому файлі реалізована вставка даних в **MongoDB**.

### 4. Реалізація в SQL базі
Ми створили структуру бази даних у **PostgreSQL**, включаючи такі сутності:
- **Проекти**
- **Пропозиції**
- **Контракти**
- **Платежі**

Використовували **збережені процедури** для маніпуляцій з даними, такі як `sp_create_project`, `sp_create_proposal`, `sp_create_message`.

#### Файл: `filling_sql.py`
Цей файл містить реалізацію вставки даних у **PostgreSQL**.

### 5. Тестування швидкодії
Ми порівняли швидкодію операцій вставки та вибірки даних у **PostgreSQL** та **MongoDB** для 10,000 записів. Результати показали, що **PostgreSQL** працює швидше при таких обсягах даних, але для більших обсягів даних **MongoDB** може показати кращі результати за рахунок горизонтального масштабування.

#### Файл: `speedcomp/sqlspeed.py`
Цей файл відповідає за тестування швидкості вставки та вибірки в **PostgreSQL**.

#### Файл: `speedcomp/nosqlspeed.py`
Цей файл відповідає за тестування швидкості вставки та вибірки в **MongoDB**.

### Опис файлів:

- **`filling_sql.py`** – Скрипт для заповнення даними SQL бази.
- **`speedcomp/nosqlspeed.py`** – Скрипт для вимірювання швидкості вставки та вибірки в MongoDB.
- **`speedcomp/sqlspeed.py`** – Скрипт для вимірювання швидкості вставки та вибірки в PostgreSQL.
- **`connectionmongodb.py`** – Файл для встановлення з'єднання з MongoDB.
- **`filling_nsql.py`** – Вставка даних в MongoDB.
- **`filling_sql.py`** – Вставка даних в PostgreSQL.

### Висновки:

- **MongoDB** продемонстрував кращу масштабованість при збільшенні обсягу даних.
- **PostgreSQL** виявився швидшим на менших обсягах даних та при виконанні складних транзакцій.
- Вибір між **NoSQL** і **SQL** базами залежить від специфіки проєкту: **NoSQL** ідеально підходить для змінюваних, неструктурованих даних, а **SQL** бази дають кращу підтримку транзакцій та структурованих даних.
